Introduction to operators

Comparison operators

  • $eq
  • $gt
  • $lt
  • $ne
  • $gte
  • $lte
Example: 
      {index:{"$eq":4}}
      {index:{"$gt":4}}
      {index:{"$lt":4}}
      {index:{"$lte":4}}
      {index:{"$gte":4}}

      chaining multiple operator condition with and.
      {index:{"$gte":4,"lte":20}}

$in and $nin operators

These are in and not in operators, basically here we have to provide values that are to be included or not included in the result in an array 

        {index:{"$in":[5,6,7]}}
        {index:{"$nin":[1,2,3]}}

$and operator

Logically combines multiple conditions resulting documents must match all conditions 

        {$and:[{condition1},{condition2},{condition3},{condition4}]

So what the difference between the , and _ and this $and?

Answer: Implicit and Explicit and

Which one to use: Explicit and when querying the field with same name, and Implicit and when querying the field with different name

basically in implicit and , the first condition will get overwritten by end

Example : db.collection_name.find({index:{"$ne":3},index:{"$gte":3}})
so we have given this condition and the value will get results with 3 also included.

$or Operator

This operator also combines multiple condition but query will look for results maching any of those conditions 

        {$or:[{condition1},{condition2},{condition3},{condition4}]

        note:if we want to us or operator in fields with same name, its better to use $in operator
